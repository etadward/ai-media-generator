## 🎬 Wan 2.7 — 「Thinking Mode」導演模型
**最適合：** 電影感運鏡 brief、首尾幀控制、畫面內多語文字。

1. **一鏡到底照順序寫：** 主體 + 動作 + 場景 + 打光 + 運鏡 + 風格。**完整句子，不是關鍵字湯** —— Thinking Mode 吃 context。
2. **丟掉 tag/權重語法**（無 `(())`、無負權重）。空間關係用白話描述（誰面對誰、誰在前），模型自己排場面。
3. **首尾幀：寫「兩幀之間的運動」**（最多人漏掉的步驟）。公式 = 起始態 + 結束態 + 過渡動作 + 運鏡行為 + 連續性註記。兩幀保持同光向、同鏡頭、同主體比例，只改一件事。
4. **兩個 toggle 別搞混：** `Thinking Mode` = 生成前推理（複雜/多主體留 ON）；`Prompt 改寫/擴寫` = LLM 灌水稀 prompt（自己寫完整 brief 後關掉，否則增延遲又漂移）。
5. **精準運動動詞：** 「slow dolly-in with handheld micro-shake」≫「dynamic camera movement」。實用細節 ~150-200 字封頂。

**範例（首尾幀產品揭幕）：** 「Start: 霧黑護膚瓶立在濕板岩陰影中。End: 同瓶前打光、標籤朝鏡頭寫『AURELIA』。過渡：瓶身朝鏡頭緩轉 1/4 圈、暖色 key light 從左升起、水珠接高光。鏡頭從桌面緩 dolly-in 到標籤高度。連續性：同瓶同板岩同淺景深。Negative: flicker, warping, blur, watermark.」

## 🔒 Kling 3.0 Omni — 物理 + 身份鎖
**最適合：** 產品/角色廣告、物理事件（水/玻璃/碎屑）、多鏡一致性。

1. **身份 vs 行為分離（核心招）：** 「長什麼樣」全放進 `@element`（2-4 個乾淨參考角度建一個），文字 prompt **只**寫發生什麼 + 運鏡。**別在文字裡重述 hero 外觀** —— 那正是漂移/變形主因。
2. **element 用 4 角度建**（正/3⁄4左/3⁄4右/背）合成一個（≤7 圖，配 3-8s 參考片則 4 圖）。每個角色給**自己的** element + 區別細節，別共用「年輕女生黑髮」否則混在一起。
3. **物理寫「事件」不寫「狀態」：**「玻璃向外炸裂、碎片飛散打滑、水花濺起」≫「破玻璃」。點名事件 + 順序，模型尊重動量/碰撞。
4. **每次都加針對性 negative：** 對症你怕的失敗（「瓶身材質不變形、標籤保持銳利、不繞圈漂移」），不是泛用 blur 清單。
5. **對白/唇同步：** `[角色, 語氣]: "台詞"`。⚠️ **一鏡一說話者最佳，~5-7s**；2+ 說話者降同步 → 分開生再剪。

**範例（i2v 產品，身份鎖 @bottle_element）：** 「Scene: 霧黑棚、單硬 key light 左斜射、薄霧。Action: 精華瓶在濕板岩上，一滴水落下擊面 → 衝擊向外漣漪、微噴霧升起後靜止、液面光澤接 key light。Camera: 低 3⁄4 角緩 dolly push 進標籤，再從水滴 rack focus 到瓶蓋。Negative: 無震動、瓶身不變形、標籤銳利可讀。」

## 🎵 HappyHorse 1.0 — 敘事 + 原生音訊
**最適合：** 需要成品配樂的一鏡敘事片。**不是**極致寫實天花板的選擇。

1. **Prompt 要短（~20-40 字），主體前置。** 固定 token 預算 → 形容詞偷算力。順序：主體+動作 → 環境+光 → **一個**運鏡。
2. **正好一個運鏡放結尾**（兩個僅限相容；絕不 3+）。具名運鏡：slow dolly-in / steadicam push / lateral orbit / crane up / locked-off。
3. **善用聯合音訊：** 聲音照發生順序寫，前景→中→背景分層。唇同步 = 台詞加引號 + 指定語言（英/普/粵/日/韓/德/法）。要靜音就明說。
4. **視覺寫，不寫情緒：**「男人獨坐空酒吧、肩前傾」not「充滿悔恨的孤獨身影」。它只渲染畫得出/發得出聲的。
5. **i2v 絕不重述靜圖** —— 只寫運動、運鏡、次要運動（蒸氣/布料）、光變、音訊。

⚠️ ~5-8s @1080p，<60 字防臉/手漂移。

## 🎞️ 拉片復刻 / 忠實復刻 — 先選模式！
**最適合：** 復刻運鏡、編舞、剪輯節奏。**敗在**一個 prompt 同時改主體+運鏡+背景。

1. **動手前先選復刻模式（#1 失敗主因）：**
   - **複製運動**（運鏡/舞蹈/打鬥）→ 動作轉移路徑（Kling 3.0 Motion Control「參考影片轉移」/ video-to-video）。
   - **複製剪輯結構**（hook/節奏/鏡序，用你的產品重填）→ 拉片/逆向工具（Seedance 2.0）逐鏡重建。
2. **明確標每個素材的角色：**「@image1 當首幀」「@video1 當運鏡參考」「@image2 當風格參考」。角色模糊 → 模型 fallback 預設、無視你的參考。
3. **鎖首幀，只描述會動的：** 匹配首幀 + 只寫運動/運鏡。加明確 hold：「定格首幀 ~0.5s，接著 slow dolly-in」。
4. **連續性鎖用白話：**「保持產品形狀、標籤顏色一致、比例不變；只改打光」。一次改一件（移除/替換/重打光/重風格）。
5. **每次只改 1 主要 + 至多 1 次要運動。** 簡化背景，逐鏡迭代。別期望像素級複製，**絕不餵版權素材想 clone**。

**範例：**「@video1 當運鏡參考，@image1 當首幀。根據首幀的產品、構圖、風格生成影片。保持產品形狀、標籤顏色一致，背景比例不變。鏡頭：定格首幀約 0.5 秒，接著緩慢推軌前進，最後輕微環繞收尾。暖色逆光、電影感對比。只改運鏡，不改主體。」

## 🧩 多模型聚合平台 meta 操作法（OiiOii 這層）
**最適合：** 重度迭代、要控成本又護 hero shot 的廣告/短影操盤手。

1. **按成本分層，不按習慣：** 便宜 workhorse 鎖構圖+運鏡 → **只**把贏的那鏡 re-run 旗艦。預算分配 ~50% 草稿 / 30% 成品 / 20% hero。
2. **用 i2v 鎖形狀不靠文字：** 先生 hero 靜圖再動。多鏡連續用 **frame chaining**（從鏡 1 匯出乾淨幀當鏡 2 首幀）—— 比文字重述更抗換模型。
3. **Prompt 不可移植，每個模型重寫：** 同字串不同結果。rendering-engine 型（Veo/Seedance）吃結構化/JSON 式；漂移型（Kling/Wan）要明確運鏡+運動約束、數值運鏡放小（1-3）。
4. **模型專屬 negative：** 漂移型上完整穩定組「morphing/warping/extra limbs/flickering/distortion/watermark/text overlay」；穩定型 negative 輕一點。
5. **換模型素材複用 gotcha：**(1) 參考圖 AR 對齊輸出 AR（否則拉伸）(2) 一次性圖參考只條件化首幀，**存成角色/資產才跨請求持續**（系列用持久 slot）(3) 注意各模型 duration/credit 預設不同（見 oiioii.md §0 時長表）。
6. **燒預算前先 benchmark：** 同 1 prompt 在 2-3 引擎用最便宜 tier 跑，比一致性/可控/運動/延遲（不是純畫質），再標準化。**這就是聚合平台的意義。**

## 📊 Shot Type → 選模型速查
| Shot type | 選 | 為什麼 |
|---|---|---|
| **乾淨產品棚拍** | HappyHorse（成品音訊一鏡）/ Kling 3.0 Omni（i2v + @element 鎖）| 主體穩 + 順 orbit/dolly；Kling 鎖形狀、HappyHorse 內建配樂 |
| **戲劇 editorial** | Wan 2.7 | Thinking Mode 從完整 brief 規劃構圖/光 + 首尾幀 |
| **角色 + 唇同步對白** | Kling 3.0 Omni（語音綁定，1 說話者/鏡）/ HappyHorse（7 語原生音訊）| 原生同步；一鏡一說話者 ~5-7s |
| **區域/參考運動控制** | Kling 3.0 Motion Control | 抽骨架/節奏/物理映射到你的主體 |
| **畫面內文字** | Wan 2.7 / Kling 3.0 Omni | 引號寫死字串 + 指定位置/材質 |
| **忠實復刻** | 運動→Kling Motion Control；剪輯結構→Seedance 2.0 | 對工具配工作，別一招打天下 |

**通用 i2v prefix（鎖形狀到靜圖）：** `根據圖片中的物體、畫面、風格來生成影片。` 後面只寫運動/運鏡，絕不重述視覺。
